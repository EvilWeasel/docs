## Windows Terminal

Windows Terminal ist eine moderne Alternative zum `conhost.exe` (Console-Host), und mit anderen Worten beschrieben: Das Fenster, in dem unsere Shell (`pwsh`, `cmd.exe`) ausgeführt werden kann.

**Vorteile**

- Unterstützt Anlegen mehrerer Profile für verschiedene Shells
- Viele Einstellungsmöglichkeiten für die Anzeige
- Hardware-Acceleration aka. Rendergeschwindigkeit
- Shortcuts für Conviniece, z.B. `strg+L` als Alias für den `clear` Command

:::tip

Am einfachsten lässt sich das Windows Terminal über den Microsoft Store installieren.

:::

## Powershell Core

In allen modernen Versionen von Microsoft Windows (seit ca. 2014) wird neben der alten `cmd.exe` Shell auch die `Windows Powershell` mitgeliefert. Es macht heutzutage wenig Sinn Scripte in `cmd.exe` zu schreiben. Powershell hat viele Vorteile gegenüber konventionellen Shells wie `bash` unter Linux.

Seit ca. Windows 10 gibt es allerdings eine neue Version von Powershell, namens `Powershell Core`. Das `Core` im Namen Signalisiert, dass diese neue Powershell Version in Dotnet Core implementiert ist, und daher Cross-Platform ist. Das heißt man kann Powershell Core auch unter Linux nutzen.

:::tip

`Powershell Core` lässt sich ebenfalls über den Microsoft Store nachinstallieren.

:::

### Git Author

Eine neue Version in Git wird als Commit bezeichnet. Ein Commit enthält alle notwendigen Informationen, welche Git benötigt, um die Version im nachhinein wiederherzustellen, wenn wir einen Fehler gemacht haben.

Außerdem enthält ein Commit immer folgende Eigenschaften:

- Datum und Uhrzeit des Commits
- Author Name
- Author Email

Während Datum und Uhrzeit von Git automatisch ausgefüllt werden können, ist klar, dass wir selbst noch die Informationen zum Author hinzufügen müssen. Dies machen wir mit dem `git-config` Command:

```bash
git config --global user.name "<dein-name>" && git config --global user.email "<deine-github-noreply-mail>"
```

:::tip

Mit deinem Github Account erhälst du zusätzlich eine "NoReply"-Email Addresse zugewiesen. Diese kannst du sicher nutzen, um deine Commits zu erstellen (damit dein Github Account gelinkt ist). Damit musst du deine private Email-Addresse nicht veröffentlichen.

:::

### SSH und Git

Die einfachste Art und Weise mit `Remote-Repositories` (wie Github) zu interagieren ist über *HTTPS*. Jedoch ist die einfachste Art meistens nicht die Beste...

Wenn wir über HTTPS mit dem Remote interagieren, müssen wir uns natürlich irgendwie Authentifizieren. Irgendwie muss der Server, also Github in unserem Beispiel, ja wissen, ob ich Berechtigt bin Änderungen zu tätigen. HTTPS ist verschlüsselt, wir können allerdings noch einen Schritt weiter gehen und die Kommunikation mit dem Server über SSH laufen lassen.

SSH steht für das **Secure-Shell** Protokoll. Was eine Shell ist, wissen wir bereits. Zur Erinnerung: Eine Shell ist ein **Read-Evaluate-Print-Loop**, oder **REPL**, mit dem wir mit dem Betriebssystem interagieren können.

Secure in dem Kontext bedeutet, dass die Verbindung über Public-Key-Exchange abgesichert wird.
